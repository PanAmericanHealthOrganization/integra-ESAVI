GUÍA DE CONFIGURACIÓN DE DESPLIEGUE: Proyecto MedraWhodrug (Tomcat / TomEE)

1. REQUISITOS DE ENTORNO
- Servidor de Aplicaciones: Apache TomEE 8.x o 9.x (Recomendado para soporte de EJB y JPA).
- Java Runtime: JDK 11.
- Base de Datos: PostgreSQL 13+ (Recomendado).

2. CONFIGURACIÓN DE LA BASE DE DATOS (DATASOURCE)
A diferencia de WildFly, en Tomcat/TomEE el DataSource se configura generalmente en el archivo `context.xml` o `server.xml`.

- Nombre del Recurso: jdbc/DsMedraWhodrug
- Nombre del JNDI: java:comp/env/jdbc/DsMedraWhodrug
- Driver: org.postgresql.Driver

Pasos para configurar en Tomcat/TomEE:
1. Colocar el driver de PostgreSQL (`postgresql-42.x.x.jar`) en la carpeta `/lib` de Tomcat.
2. Editar el archivo `conf/context.xml` o crear uno en `META-INF/context.xml` dentro del proyecto con el siguiente contenido:

<Resource name="jdbc/DsMedraWhodrug" 
           auth="Container" 
           type="javax.sql.DataSource" 
           driverClassName="org.postgresql.Driver" 
           url="jdbc:postgresql://<host>:<puerto>/<nombre_base_datos>" 
           username="<usuario_db>" 
           password="<contraseña_db>" 
           maxTotal="20" 
           maxIdle="10" 
           maxWaitMillis="-1"/>

3. Asegurarse de que la clase `Utils.java` y la unidad de persistencia en `persistence.xml` apunten al nombre del DataSource configurado.

3. CONFIGURACIÓN DE CARGA DE ARCHIVOS (EXCEL/CSV)
El código actual de `ExcelReaderSingleton.java` busca la propiedad de sistema `jboss.home.dir`. Para que funcione en Tomcat, se debe realizar una de las siguientes acciones:

A. Modificar el código para usar `catalina.home` en lugar de `jboss.home.dir`.
B. O, añadir la propiedad de sistema al arranque de Tomcat en el archivo `setenv.sh` o `setenv.bat`:
   - CATALINA_OPTS="-Djboss.home.dir=/ruta/a/tomcat"

Rutas de carpetas a crear en la raíz de Tomcat:
- Datos de MedDRA: ${catalina.home}/data-medra/
- Datos de WhoDrug: ${catalina.home}/data-whodrug/

4. PASOS DE DESPLIEGUE
1. Instalar el Driver de PostgreSQL en `/lib`.
2. Configurar el Resource en `context.xml`.
3. Crear las carpetas `data-medra` y `data-whodrug` en la raíz de la instalación de Tomcat.
4. Colocar los archivos de datos en sus respectivas carpetas.
5. Desplegar el archivo .war en la carpeta `webapps`.
6. Iniciar el servidor.

5. NOTAS IMPORTANTES SOBRE COMPATIBILIDAD
- EJB y JPA: Si se utiliza Tomcat "estándar" (no TomEE), las anotaciones `@Singleton`, `@Startup` y `@PersistenceContext` no funcionarán automáticamente. Se requerirá la inclusión de librerías como Hibernate y un gestor de dependencias como Spring o la migración a TomEE.
- JAX-RS: El despliegue de la API REST requiere que el contenedor soporte la especificación JAX-RS (incluida en TomEE).

6. VERIFICACIÓN
- Revisar los logs en `logs/catalina.out` para verificar que el `ExcelReaderSingleton` haya procesado los archivos.
- Probar la API en: http://<host>:<puerto>/MedraWhodrug/api/whodrug/abbreviation
